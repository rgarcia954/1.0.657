uint32_t  reg 
)

Apply register value to flash control register.

Parameters
[in]flashflash instance being used
[in]regvalue to apply to flash IF_CTRL register
Note
Changing VREAD1_MODE, RECALL or LP_MODE issues a command to the flash interface. As these bits are mutually exclusive and only one command can be executed at time, this function applies the register values three time to ensure all bit changes are applied.

__STATIC_INLINE uint32_t Sys_Flash_ReadRegDelayCTRL3 ( const FLASH_Type *  flash)

Read delay control register no 3.

Note
Delay control register provides erase time for sector, max 4 erase pulse required
When FLASH_DELAY_CTRL register is written based on the core clock a default erase time of 0.9ms is applied.
This delay must be multiplied by 4, when using only a single retry pulse of FLASH_RETRY_4 for sector erase.

)

Write a 38-bit word to a flash instance.

This function writes a 38-bit word to a flash instance

Parameters
[in]flashflash instance to be used
[in]addraddress to be written belonging to the flash instance provided
[in]word32-bit C pointer to the word to be stored in flash,
word[0] [0:31] bits consist of first lower 32-bits
word[1] [0:5] bits consist of the remaining higher 6-bits
[in]enb_enduranceset to 0 for default flash endurance
set to 1 to enable two stage programming for higher endurance of data programmed
Returns
Flash API status code

FlashStatus_t
Note
System clock frequency should not be changed when flash is being programmed
